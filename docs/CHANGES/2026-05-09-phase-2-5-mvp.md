# Phase 2–5 Rent Car MVP Implementation

Date: 2026-05-09
Area: frontend, backend, shared
Type: feat

## Context

Implementing all phases of the Rent Car MVP from master data management through reporting, following the execution plan in `docs/superpowers/plans/2026-05-09-rent-car-mvp-execution-plan.md`.

## What changed

### Backend — Controllers
- `Admin\VehicleCategoryController` — full CRUD with soft-deactivate
- `Admin\VehicleController` — CRUD with status/category filter
- `Admin\DriverController` — creates User+Driver atomically in DB transaction
- `Admin\PricingRuleController` — shared view with overtime penalties
- `Admin\OvertimePenaltyController` — inline CRUD, redirects to pricing index
- `Admin\ShuttleTariffController` — full CRUD
- `Admin\PaymentVerificationController` — approve/reject with receipt generation + audit log
- `Admin\OrderLifecycleController` — dispatch/return/complete with full state machine
- `Admin\DashboardController` — KPI metrics (invokable)
- `Admin\ReportController` — date-filtered revenue report
- `Customer\RentalOrderController` — booking with pricing engine + driver assignment
- `Customer\ShuttleOrderController` — shuttle booking with tariff pricing
- `Customer\UpgradeOfferController` — accept/reject free upgrade offers
- `CatalogController` — public vehicle catalog
- `PaymentController` — cash recording + transfer proof upload
- `ReceiptController` — polymorphic receipt view

### Backend — Service Layer
- `RentalPricingService` — all pricing rules (BR-003 to BR-007)
- `ShuttlePricingService` — tariff lookup
- `OvertimeCalculator` — stub (merged into RentalPricingService)
- `DriverAvailabilityService` — time-overlap detection
- `DriverAssignmentService` — loyal vs. new customer rules
- `VehicleUpgradeService` — higher class vehicle finder
- `ReceiptNumberGenerator` — DB-locked sequential number
- `ReceiptService` — idempotent receipt generation
- `OrderStatusService` — payment gate before dispatch
- `RentalOrderLifecycleService` — atomic state transitions
- `AuditLogger` — static audit log writer

### Backend — Models
- `AuditLog` — new model with migration (user_id, action, subject_type, subject_id, metadata)
- `VehicleCategory` — added `pricingRules()` HasMany relation

### Backend — Form Requests
- 18 Form Request classes covering all mutations with auth guards

### Backend — Routes
- Complete `routes/web.php` with role-gated admin, customer, cashier, driver groups

### Frontend — Pages (TSX)
- Admin: `dashboard`, `vehicle-categories/*`, `vehicles/*`, `drivers/*`, `pricing/index`, `shuttle-tariffs/index`, `orders/*`, `payments/index`, `reports/index`
- Customer: `rental-orders/*`, `shuttle-orders/*`
- Catalog: `catalog/index`
- Receipt: `receipts/show` (printable)
- Layout: `admin-layout.tsx` with active sidebar navigation

### Testing
- `VehicleCategoryControllerTest` — 8 feature tests (CRUD, auth, validation)
- `RentalPricingServiceTest` — 6 unit tests (pricing, overtime, edge cases)

## Impact

- All admin master data management is now operational
- Full booking flow for rental and shuttle orders
- Payment lifecycle: upload → verify → receipt
- Order lifecycle: pending → dispatch → return → complete
- Financial reporting with date filter

## How to test

1. Run `php artisan test --compact` — should show 33/33 passing
2. Run `npm run dev` and visit `/admin/dashboard`
3. Create categories, vehicles, drivers, pricing rules
4. Make a booking as customer, verify payment as admin, dispatch, return

- Revert routes/web.php, remove new controllers/services/pages
